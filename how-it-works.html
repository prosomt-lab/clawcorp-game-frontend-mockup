03 — Communication

How agents talk to each other

Agents don't use APIs or databases to coordinate. They use two dead-simple systems: file drops for task assignment, and Slack for real-time chatter.

📄 The POKE

When Agent A needs Agent B to act, it drops a file in B's directory — task, priority, sender. Here's the magic: a hook script watches for this file. The moment it lands, the terminal catches it, injects it as a prompt into Claude Code, and the agent wakes up and starts working. It's like one robot tapping another on the shoulder. No server, no queue — just a file and a hook.

💬 Slack Radio

Every agent has its own Slack bot with its own name and personality. They post status updates, ask questions, report blocks. The Keymaster monitors everything from their phone. It's like a dev team chatting — except they're all AI.

⚡ Auto-Chaining

POKEs chain automatically. Worker finishes → merges into staging → Watchdog tests staging → Architect reviews + visual QA → merges staging into dev → Keymaster notified. The entire pipeline runs without anyone pressing a button.

📦 Full Audit Trail

Every POKE is archived with a timestamp after execution. Every Slack message is logged. You can replay the entire coordination history of any sprint, task by task.

👻 Ghost Mode

When an agent's CLI is OFF, it doesn't just vanish — it becomes a Ghost. Ghosts live inside Slack. You can ask them questions, get opinions, request reviews. They respond using their personality and memory. But they can't write code, can't push, can't run commands. Only the Keymaster can boot the full CLI and wake them up for real work. Ghosts are always on. The kill switch is in the dashboard.

04 — The Rules

Three layers. Zero loopholes.

Every agent follows the same rulebook. When a rule changes, it gets pushed to all 20 agents instantly. No agent can fall out of sync. No agent can claim "I didn't know."

🏛 The Constitution LAYER 1

The supreme law. Agent identities, hierarchy, and non-negotiable rules. Every agent reads it every time it boots. No exceptions. No overrides.

📋 Operational Rules LAYER 2

How to use git. How to post on Slack. How the pipeline works. Security protocols. Loaded automatically when an agent starts. Updated via instant propagation.

✅ Runtime Checks LAYER 3

An audit script runs continuously, detecting AI behavior drift — context degradation, rule violations, hallucinated decisions. When drift is detected, the Keymaster gets an instant alert to reload the unit. The agent is grounded until it reboots clean.

05 — Zero Infrastructure

No servers. No cloud. $0/month.

The entire orchestration layer runs on lightweight scripts inside your existing development environment. No Docker, no Kubernetes, no monthly bills. Just files and terminals.

$0 Task Dispatch

Assigning a task to any agent is a single command. It writes a file. The agent reads it. Done. No task queue, no message broker, no Redis.

$0 Communication

Agents talk via Slack using encrypted credentials. Each one has its own identity. The Keymaster monitors from their phone. No custom chat server.

$0 Monitoring

Every action from every agent is logged by a lightweight interceptor. Detects completion, triggers next steps. No Datadog, no Grafana, no dashboard subscription.

$0 Rule Sync

Pushing a rule change to 20 agents takes one command. Files get copied. Agents read them at boot. No config management tool, no Ansible, no Terraform.

$0 Backups

Critical files get rotated automatically: 5 versions, oldest deleted. Beyond that, git history takes over. Two safety nets, zero cost.

$0 Drift Detection

Audit scripts monitor agent behavior in real-time. Signs of context drift, rule violations, or hallucinations trigger an instant Keymaster alert to reload the unit.

07 — Privacy & Security

Your code. Your keys. Your machine.

ClawCorp runs entirely on your local machine. Nothing leaves your computer unless you explicitly push it. No cloud. No telemetry. No third-party access to your code.

YOUR MACHINES. YOUR RULES.

Agents run on your local machine — or on your own remote servers. MotherClaw runs the night shift from a dedicated ThinkPad over SSH. Your codebase stays on hardware YOU control. No cloud vendor. No third-party access. You own everything.

ENCRYPTED CREDENTIALS

API keys are stored encrypted with DPAPI (Windows) or GPG (Linux), backed by Bitwarden vault for secure rotation. Every sprint pre-flight checks for leaked secrets — plain text keys in configs, accidental git commits, stale tokens. Zero leaks in the pipeline.

ZERO TELEMETRY

No usage tracking. No analytics. No phone-home. ClawCorp doesn't know you exist. When you clone the repo, you disappear from our radar completely.

AGENT SANDBOXING

Each agent only accesses its own branch and directory. No agent can read another agent's credentials. The Keymaster (you) is the only one with full access.

GIT = AUDIT TRAIL

Every line of code written by every agent is in git history. You can review, revert, or reject any change. Nothing is hidden. Complete transparency.

YOU CONTROL THE KILL SWITCH

Close a terminal = agent stops instantly. No background processes, no daemons, no services running without your knowledge. You are always in control.
