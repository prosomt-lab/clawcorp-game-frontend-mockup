Sprint Pipeline

From GO to SHIPPED.

Every sprint follows this exact flow. The POKE system chains each step automatically. Zero manual coordination after the Keymaster says GO.

01

Keymaster says GO

Sprint scope is defined. Architecte receives the green light.

02

Architecte dispatches

Arch decomposes the sprint into tasks and POKEs each worker terminal.

POKE → Forge POKE → Backbone POKE → Pixel POKE → Echo
03

Workers code in parallel

Each worker codes on their own branch. Workers update their tasks in the board with status changes and solution notes — what they built, which files they touched. The Keymaster can drop into any terminal at any time.

# Each worker = 1 branch = 1 terminal = 1 board task
Forge    → branch forge    → board: InProgress → Done + comment
Backbone → branch backbone → board: InProgress → Done + comment
Pixel    → branch pixel    → board: InProgress → Done + comment
Echo     → branch echo     → board: InProgress → Done + comment
04

Last worker done → Watchdog wakes up

The audit hook detects the last git push. Watchdog is automatically POKEd to start pre-merge testing on every branch.

AUTO • hook detects last push POKE → Watchdog
05

Watchdog pre-merge QA

Tests each branch individually. If a branch fails: Watchdog POKEs the worker who broke it. They fix, push, Watchdog re-tests. Loop until green.

PASS → POKE Architecte FAIL → POKE worker fautif
06

Architecte merges into dev

All branches green. Arch reads each task's board comments to spot file overlaps, then merges everything into dev and launches localhost.

Read board comments → detect overlaps
07

Arch fans out QA POKEs

Dev merged and localhost running. Arch POKEs all QA actors in parallel:

POKE → Watchdog (re-test dev) POKE → Eye (visual QA) POKE → Manager (audit) POKE → Keymaster (ready for review)
08

Post-merge QA (parallel)

Watchdog, Eye, Manager, and Keymaster all review the merged dev branch simultaneously. Meanwhile, the Architecte starts prepping the next sprint.

Arch preps next sprint during QA
09

QA confirmed → Workers sync

Keymaster or Eye confirms QA. Architecte auto-POKEs all workers: pull dev, type "READY FOR NEXT SPEEDRUN" in their terminal.

POKE → All workers (pull dev) Workers type: READY FOR NEXT SPEEDRUN
10

Keymaster decides

All workers synced and ready. The Keymaster gives the final call:

GO main → ship to production GO next sprint → Arch dispatches again
The POKE System

A file drops. A terminal wakes up.

POKE is how agents communicate. No API calls. No server. Just a markdown file written to a worker's directory. When the worker boots, it reads its POKE and executes.

# Architecte POKEs Forge to build a WebSocket handler

write-poke.ps1 -Agent "forge" -Orders "Implement WebSocket handler for real-time updates. Branch: forge. ETA: 15min." -From "architecte"

# This creates:
#   workers/forge/POKE.md
#
# Contents:
#   **FROM:** architecte
#   **PRIORITY:** P1
#   **ORDERS:** Implement WebSocket handler for real-time updates.
#
# When Forge boots, it reads POKE.md, executes, then archives it.

üìÑ POKE = a file

A markdown file dropped into a worker's directory. Contains orders, priority, and sender. The worker reads it on boot and executes. Then archives it. That's it.

ü™ù HOOK = an interceptor

Every time a worker runs a command, edits a file, or pushes code, a hook fires. Invisible. The audit hook logs everything. Pattern-matches for drift. Triggers POKEs automatically.

üí¨ SLACK = the radio

Each agent has its own Slack bot identity. They post status, ask questions, report blocks. The Keymaster sees everything on mobile. Every message has a sender tag — you always know who's talking.

Zero-Cost Glue

PowerShell scripts. $0/month.

The entire orchestration layer runs on PowerShell scripts inside existing VSCode terminals. No servers. No containers. No subscriptions. Just files and shell commands.

$0 write-poke.ps1

Writes a POKE.md file into any worker's directory. The receiving worker reads it on boot and executes the orders. One script replaces an entire task dispatch system.

$0 archive-poke.ps1

After a POKE is executed, this script moves it to the archive. Clean directory, audit trail preserved. Zero manual cleanup.

$0 slack-post.ps1

Posts to any Slack channel as any bot identity. DPAPI-encrypted credentials. Each agent has its own bot persona. The Keymaster always knows who's talking.

$0 audit hook

Intercepts every action from every worker. Bash commands, file edits, git pushes — all logged. Detects when the last worker finishes and auto-POKEs Watchdog. The invisible backbone of the pipeline.

FUTURE Smart audit

Pattern-match worker behavior in real-time. Detect drift, off-topic work, infinite loops. Auto-POKE the Architecte with an alert. The Manager role becomes optional — replaced by 20 lines of script logic.

$0 Slack identity

When the Architecte speaks on Slack, you see exactly which clone it is. [ARCH:clone-1] vs [ARCH:ghost] vs [ARCH:terminal-3]. Full transparency. No ambiguity.

The Manager Question

Today: an agent. Tomorrow: a script.

The Manager agent currently monitors all workers during sprints. It reads audit logs, watches for drift, and alerts the Architecte. But everything it does can be automated.

Today

Manager = an AI agent that:

• Reads audit hook logs
• Detects off-topic work
• Alerts Arch via Slack
• Consults Keymaster when unsure
• Costs tokens per session

Tomorrow

Manager = 20 lines of PowerShell that:

• Same audit hook, smarter patterns
• Auto-POKE Arch on drift detection
• Arch decides: stop, discuss, relaunch, or escalate to Keymaster
• Zero tokens. Zero agent. Pure script.
• Manager agent freed for side quests only
